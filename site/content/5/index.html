    3.3 Installation und erste Schritte mit Git¶

    Einführung¶

    In diesem Kapitel lernen Sie, wie Sie Git auf Ihrem System installieren und die ersten Schritte ausführen.

    Installation¶

    Windows¶

    Methode 1: Git für Windows (empfohlen)

    1. Download: Besuchen Sie https://git-scm.com/download/win
    2. Installation: Führen Sie den Installer aus
    3. Konfiguration: Wählen Sie die gewünschten Optionen
    4. Git from the command line and also from 3rd-party software (empfohlen)
    5. Use the OpenSSL library
    6. Checkout Windows-style, commit Unix-style line endings
    7. Use Windows' default console window

    Methode 2: Chocolatey (für erfahrene Nutzer)

    1
2
3
4
5
    # Chocolatey installieren (falls nicht vorhanden)
Set-ExecutionPolicy Bypass -Scope Process -Force; [System.Net.ServicePointManager]::SecurityProtocol = [System.Net.ServicePointManager]::SecurityProtocol -bor 3072; iex ((New-Object System.Net.WebClient).DownloadString('https://community.chocolatey.org/install.ps1'))

# Git installieren
choco install git

    macOS¶

    Methode 1: Git für macOS

    1. Download: Besuchen Sie https://git-scm.com/download/mac
    2. Installation: Führen Sie den Installer aus

    Globale Konfiguration¶

    Warum konfigurieren? - Git benötigt Ihre Identität für Commits - Konfiguration wird für alle Projekte verwendet

    # Name festlegen (wird in Commits angezeigt)
git config --global user.name "Max Mustermann"

# E-Mail festlegen (wird in Commits verwendet)
git config --global user.email "max.mustermann@example.com"

# Standard-Editor festlegen (für Commit-Nachrichten)
git config --global core.editor "code --wait"  # VS Code
# oder
git config --global core.editor "vim"          # Vim
# oder
git config --global core.editor "nano"         # Nano

# Standard-Branch-Name festlegen (main statt master)
git config --global init.defaultBranch main

2
3

git reset --hard       # Alle Änderungen verwerfen (Vorsicht!)

    Zusammenfassung¶

    Installation: - Windows: Git für Windows oder Chocolatey - macOS: Git für macOS oder Homebrew - Linux: Paketmanager (apt, dnf)

    Konfiguration: - Globale Konfiguration: git config --global - Name und E-Mail sind erforderlich - Editor kann konfiguriert werden

    Erste Schritte: 1. git init - Repository erstellen 2. Datei erstellen 3. git add - Zur Staging Area hinzufügen 4. git commit - Commit erstellen 5. git status / git log - Status prüfen

    Wichtige Befehle: - git status - Status anzeigen - git add - Änderungen hinzufügen - git commit - Commit erstellen - git diff - Unterschiede anzeigen - git log - Historie anzeigen

    Installation und erste Schritte

    Basierend auf der Installation und ersten Schritten mit Git:

    1. Erklären Sie den Unterschied zwischen globaler und projekt-spezifischer Git-Konfiguration mit einem konkreten Beispiel
    2. Beschreiben Sie den Prozess von einer neuen Datei bis zum ersten Commit mit allen Befehlen
    3. Erklären Sie, warum man git status regelmäßig verwenden sollte und was die verschiedenen Statusmeldungen bedeuten
    \[\text{Beispiel: Ein Entwickler arbeitet an zwei verschiedenen Projekten...}\]
    Tipp

    Erstellen Sie ein Skript, das git status automatisch nach jedem Befehl ausführt

    Lösung
    Lösungsvideo

    1. Globale vs. projekt-spezifische Konfiguration: Globale Konfiguration (git config --global): - Wird für alle Projekte auf dem Computer verwendet - Enthält typischerweise: Name, E-Mail, Standard-Editor - Beispiel: git config --global user.name "Max Mustermann"

    Projekt-spezifische Konfiguration (git config ohne --global): - Gilt nur für das aktuelle Projekt - Nützlich für: Projektspezifische E-Mail, spezielle Editor-Einstellungen - Beispiel: Ein Freelancer hat unterschiedliche E-Mails für verschiedene Kunden

    Konkretes Beispiel: 

    1
2
3
4
5
6
7
8
9
    Globale Konfiguration:
user.name = Max Mustermann
user.email = max@privat.de

Projekt-spezifisch (in Kundenprojekt):
user.email = max@kunde1.de

Projekt-spezifisch (in Firmenprojekt):
user.email = max@firma.de

    2. Prozess bis zum ersten Commit: 

    1
2
3
4
5
6
    1. Datei erstellen: echo "Inhalt" > datei.txt
2. Status prüfen: git status → "Untracked files"
3. Zur Staging Area hinzufügen: git add datei.txt
4. Status prüfen: git status → "Changes to be committed"
5. Commit erstellen: git commit -m "Erste Nachricht"
6. Historie prüfen: git log

    3. Warum git status regelmäßig verwenden: - Übersicht: Zeigt aktuellen Zustand des Projekts - Fehlervermeidung: Verhindert falsche Commits - Statusmeldungen: - Untracked files: Neue Dateien, noch nicht versioniert - Changes not staged for commit: Geänderte Dateien, nicht bereitgestellt - Changes to be committed: Bereitgestellte Änderungen, bereit für Commit - nothing to commit: Alle Änderungen sind committet

    Analogie: git status ist wie ein Spiegel vor dem Verlassen des Hauses - prüft, ob alles korrekt ist.
