# 3.3 Installation und erste Schritte mit Git

## Einführung

In diesem Kapitel lernen Sie, wie Sie Git auf Ihrem System installieren und die ersten Schritte ausführen.

## Installation

### Windows

**Methode 1: Git für Windows (empfohlen)**

1. **Download**: Besuchen Sie https://git-scm.com/download/win
2. **Installation**: Führen Sie den Installer aus
3. **Konfiguration**: Wählen Sie die gewünschten Optionen
   - `Git from the command line and also from 3rd-party software` (empfohlen)
   - `Use the OpenSSL library`
   - `Checkout Windows-style, commit Unix-style line endings`
   - `Use Windows' default console window`

**Methode 2: Chocolatey (für erfahrene Nutzer)**

```powershell
# Chocolatey installieren (falls nicht vorhanden)
Set-ExecutionPolicy Bypass -Scope Process -Force; [System.Net.ServicePointManager]::SecurityProtocol = [System.Net.ServicePointManager]::SecurityProtocol -bor 3072; iex ((New-Object System.Net.WebClient).DownloadString('https://community.chocolatey.org/install.ps1'))

# Git installieren
choco install git
```

### macOS

**Methode 1: Git für macOS**

1. **Download**: Besuchen Sie https://git-scm.com/download/mac
2. **Installation**: Führen Sie den Installer aus

**Methode 2: Homebrew (empfohlen)**

```bash
# Homebrew installieren (falls nicht vorhanden)
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# Git installieren
brew install git
```

### Linux (Ubuntu/Debian)

```bash
# Aktualisieren der Paketliste
sudo apt update

# Git installieren
sudo apt install git
```

### Linux (Fedora/RHEL)

```bash
# Git installieren
sudo dnf install git
```

## Erste Konfiguration

### Globale Konfiguration

**Warum konfigurieren?**
- Git benötigt Ihre Identität für Commits
- Konfiguration wird für alle Projekte verwendet

**Befehle**:

```bash
# Name festlegen (wird in Commits angezeigt)
git config --global user.name "Max Mustermann"

# E-Mail festlegen (wird in Commits verwendet)
git config --global user.email "max.mustermann@example.com"

# Standard-Editor festlegen (für Commit-Nachrichten)
git config --global core.editor "code --wait"  # VS Code
# oder
git config --global core.editor "vim"          # Vim
# oder
git config --global core.editor "nano"         # Nano

# Standard-Branch-Name festlegen (main statt master)
git config --global init.defaultBranch main
```

**Konfiguration überprüfen**:

```bash
# Alle Konfigurationen anzeigen
git config --list

# Spezifische Konfiguration anzeigen
git config user.name
git config user.email
```

### Projekt-spezifische Konfiguration

**Für einzelne Projekte**:

```bash
# In das Projektverzeichnis wechseln
cd /pfad/zum/projekt

# Projekt-spezifische Konfiguration
git config user.name "Projekt-spezifischer Name"
git config user.email "projekt@example.com"
```

## Erste Schritte mit Git

### Schritt 1: Neues Repository erstellen

**Methode 1: Neues Projekt**

```bash
# Projektverzeichnis erstellen
mkdir mein-projekt
cd mein-projekt

# Git Repository initialisieren
git init
```

**Ausgabe**:
```
Initialized empty Git repository in /pfad/zum/projekt/.git/
```

**Methode 2: Bestehendes Projekt**

```bash
# In bestehendes Projektverzeichnis wechseln
cd /pfad/zum/bestehenden-projekt

# Git Repository initialisieren
git init
```

### Schritt 2: Erste Datei erstellen und committen

**1. Datei erstellen**:

```bash
# Erstelle eine README-Datei
echo "# Mein erstes Git Projekt" > README.md

# Oder mit einem Texteditor
code README.md  # VS Code
# oder
notepad README.md  # Windows
```

**2. Status prüfen**:

```bash
git status
```

**Ausgabe**:
```
On branch main

No commits yet

Untracked files:
  (use "git add <file>..." to include in what will be committed)
        README.md

nothing added to commit but untracked files present (use "git add" to track)
```

**3. Datei zur Staging Area hinzufügen**:

```bash
# Einzelne Datei
git add README.md

# Oder alle Dateien
git add .
```

**4. Status erneut prüfen**:

```bash
git status
```

**Ausgabe**:
```
On branch main

No commits yet

Changes to be committed:
  (use "git rm --cached <file>..." to unstage)
        new file:   README.md
```

**5. Commit erstellen**:

```bash
git commit -m "Initial commit: Add README"
```

**Ausgabe**:
```
[main (root-commit) abc1234] Initial commit: Add README
 1 file changed, 1 insertion(+)
 create mode 100644 README.md
```

### Schritt 3: Änderungen verwalten

**1. Änderungen an README.md**:

```bash
echo "## Projektbeschreibung" >> README.md
echo "Dies ist ein Testprojekt für Git." >> README.md
```

**2. Status prüfen**:

```bash
git status
```

**Ausgabe**:
```
On branch main
Changes not staged for commit:
  (use "git add <file>..." to update what will be committed)
  (use "git restore <file>..." to discard changes in working directory)
        modified:   README.md

no changes added to commit (use "git add" and/or "git commit -a")
```

**3. Änderungen anzeigen**:

```bash
git diff
```

**Ausgabe**:
```
diff --git a/README.md b/README.md
index e69de29..d95f3ad 100644
--- a/README.md
+++ b/README.md
@@ -0,0 +1,2 @@
+# Mein erstes Git Projekt
+## Projektbeschreibung
```

**4. Änderungen hinzufügen und committen**:

```bash
git add README.md
git commit -m "Add project description"
```

### Schritt 4: Projektstruktur erstellen

**1. Verzeichnisse erstellen**:

```bash
mkdir src docs tests
```

**2. Dateien in Verzeichnissen**:

```bash
echo "// JavaScript Code" > src/main.js
echo "# Dokumentation" > docs/README.md
echo "import unittest" > tests/test_main.py
```

**3. Alle Änderungen hinzufügen**:

```bash
git add .
```

**4. Commit erstellen**:

```bash
git commit -m "Add project structure: src, docs, tests"
```

## Wichtige Befehle für den Anfang

### Status-Übersicht

```bash
git status          # Zeigt aktuellen Status
git log             # Zeigt Commit-Historie
git log --oneline   # Komprimierte Historie
```

### Änderungen verwalten

```bash
git add <file>      # Einzelne Datei hinzufügen
git add .           # Alle Änderungen hinzufügen
git diff            # Unterschiede anzeigen
git diff --staged   # Unterschiede in Staging Area
```

### Commits

```bash
git commit -m "Nachricht"          # Commit mit Nachricht
git commit --amend                 # Letzten Commit ändern
git commit --amend --no-edit       # Letzten Commit ohne neue Nachricht
```

### Fehlerbehebung

```bash
git restore <file>     # Änderungen verwerfen (Working Directory)
git restore --staged <file>  # Aus Staging Area entfernen
git reset --hard       # Alle Änderungen verwerfen (Vorsicht!)
```

## Praktische Übung

### Übung 1: Erstes Repository

```bash
# 1. Neues Projekt erstellen
mkdir git-uebung
cd git-uebung

# 2. Git initialisieren
git init

# 3. README erstellen
echo "# Git Übungsprojekt" > README.md

# 4. Erster Commit
git add README.md
git commit -m "Initial commit"

# 5. Weitere Dateien erstellen
echo "console.log('Hallo Welt');" > script.js
echo "body { color: blue; }" > style.css

# 6. Zweiter Commit
git add .
git commit -m "Add basic files"

# 7. Historie anzeigen
git log --oneline
```

### Übung 2: Änderungen verfolgen

```bash
# 1. README ändern
echo "## Lernziele" >> README.md
echo "- Git verstehen" >> README.md

# 2. Status prüfen
git status

# 3. Diff anzeigen
git diff

# 4. Commit erstellen
git add README.md
git commit -m "Add learning objectives"

# 5. Historie prüfen
git log --oneline
```

## Zusammenfassung

**Installation**:
- Windows: Git für Windows oder Chocolatey
- macOS: Git für macOS oder Homebrew
- Linux: Paketmanager (apt, dnf)

**Konfiguration**:
- Globale Konfiguration: `git config --global`
- Name und E-Mail sind erforderlich
- Editor kann konfiguriert werden

**Erste Schritte**:
1. `git init` - Repository erstellen
2. Datei erstellen
3. `git add` - Zur Staging Area hinzufügen
4. `git commit` - Commit erstellen
5. `git status` / `git log` - Status prüfen

**Wichtige Befehle**:
- `git status` - Status anzeigen
- `git add` - Änderungen hinzufügen
- `git commit` - Commit erstellen
- `git diff` - Unterschiede anzeigen
- `git log` - Historie anzeigen
