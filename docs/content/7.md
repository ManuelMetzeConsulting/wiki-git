# 4.2 GitHub/GitLab Integration

## Einführung

GitHub und GitLab sind die beiden größten Plattformen für Git-Repositories. Sie bieten weit mehr als nur Speicherplatz - sie sind komplette Entwicklungsumgebungen.

## GitHub vs. GitLab

### GitHub

**Eigenschaften**:
- **Größte Plattform**: Über 100 Millionen Entwickler
- **Microsoft-Eigentum**: Seit 2018
- **Fokus**: Open Source, Community
- **Kosten**: Kostenlos für öffentliche Repositories, private mit Einschränkungen

**Besonderheiten**:
- GitHub Actions (CI/CD)
- GitHub Pages (Hosting)
- GitHub Copilot (KI-Assistent)
- GitHub Discussions
- GitHub Marketplace

### GitLab

**Eigenschaften**:
- **Selbst gehostet möglich**: On-Premise Installation
- **DevOps-Plattform**: Komplette CI/CD Pipeline
- **Fokus**: Enterprise, DevOps
- **Kosten**: Kostenlos für öffentliche Repositories, Enterprise-Lizenzen

**Besonderheiten**:
- Integrierte CI/CD (`.gitlab-ci.yml`)
- Container Registry
- Kubernetes Integration
- Security Scanning
- Self-Hosted Option

## GitHub einrichten

### Konto erstellen

1. **Besuchen**: https://github.com
2. **Registrieren**: E-Mail, Passwort, Benutzername
3. **E-Mail bestätigen**: Link in Bestätigungsmail klicken
4. **Profil vervollständigen**: Name, Bio, Avatar

### SSH-Schlüssel einrichten

**Warum SSH?**
- Sicherer als HTTPS
- Kein Passwort bei jedem Push nötig
- Besser für Automatisierung

**Schlüssel generieren**:

```bash
# SSH-Schlüssel generieren (RSA)
ssh-keygen -t rsa -b 4096 -C "ihre@email.de"

# Oder mit Ed25519 (empfohlen)
ssh-keygen -t ed25519 -C "ihre@email.de"

# Ort: ~/.ssh/id_rsa oder ~/.ssh/id_ed25519
```

**Schlüssel anzeigen**:

```bash
# Linux/macOS
cat ~/.ssh/id_ed25519.pub

# Windows (PowerShell)
type $env:USERPROFILE\.ssh\id_ed25519.pub
```

**Schlüssel zu GitHub hinzufügen**:
1. GitHub Einstellungen → SSH and GPG keys
2. New SSH key
3. Titel: "Mein Computer"
4. Key: Inhalt der .pub-Datei einfügen
5. Add SSH key

**SSH-Verbindung testen**:

```bash
ssh -T git@github.com
# Erfolg: "Hi username! You've successfully authenticated..."
```

### Repository erstellen

**Methode 1: Über GitHub-Oberfläche**

1. GitHub starten → New repository
2. Repository-Name: `mein-projekt`
3. Beschreibung: Kurze Beschreibung
4. Öffentlich/Privat wählen
5. README hinzufügen (optional)
6. .gitignore hinzufügen (optional)
7. License wählen (optional)
8. Create repository

**Methode 2: Von der Kommandozeile**

```bash
# 1. Lokales Repository vorbereiten
cd /pfad/zum/projekt
git init
git add .
git commit -m "Initial commit"

# 2. Repository auf GitHub erstellen (über API)
# Oder manuell auf GitHub erstellen und dann verbinden

# 3. Remote hinzufügen
git remote add origin git@github.com:username/repo-name.git

# 4. Push
git push -u origin main
```

## GitHub Actions

### Was ist GitHub Actions?

**Definition**: CI/CD-Plattform von GitHub für Automatisierung

**Anwendungsfälle**:
- Automatisches Testen
- Code-Review
- Deployment
- Security Scanning
- Release-Erstellung

### Workflow-Struktur

**Datei**: `.github/workflows/main.yml`

```yaml
name: CI Pipeline

on:
  push:
    branches: [ main ]
  pull_request:
    branches: [ main ]

jobs:
  test:
    runs-on: ubuntu-latest

    steps:
    - uses: actions/checkout@v4

    - name: Setup Python
      uses: actions/setup-python@v5
      with:
        python-version: '3.11'

    - name: Install dependencies
      run: |
        python -m pip install --upgrade pip
        pip install -r requirements.txt

    - name: Run tests
      run: |
        python -m pytest
```

### Beispiele

**1. Automatisches Testen**:

```yaml
name: Test

on: [push, pull_request]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
    - uses: actions/checkout@v4
    - name: Run tests
      run: |
        npm install
        npm test
```

**2. Deployment**:

```yaml
name: Deploy to GitHub Pages

on:
  push:
    branches: [ main ]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
    - uses: actions/checkout@v4
    - name: Build
      run: |
        npm install
        npm run build
    - name: Deploy
      uses: peaceiris/actions-gh-pages@v3
      with:
        github_token: ${{ '{{' }} secrets.GITHUB_TOKEN {{ '}}' }}
        publish_dir: ./dist
```

## GitHub Pages

### Was ist GitHub Pages?

**Definition**: Kostloser Hosting-Service für statische Websites

**Anwendungsfälle**:
- Dokumentation
- Portfolio
- Projekt-Websites
- Blogs

### Einrichtung

**Methode 1: Repository als GitHub Pages**

1. Repository erstellen: `username.github.io`
2. Index.html erstellen
3. Push zum `main` Branch
4. Website ist unter `https://username.github.io` verfügbar

**Methode 2: Dokumentation in Repository**

1. Dokumentation in `/docs` Ordner
2. GitHub Settings → Pages
3. Source: `main` Branch, `/docs` Ordner
4. Website ist unter `https://username.github.io/repo-name` verfügbar

**Beispiel für MkDocs**:

```yaml
# .github/workflows/deploy.yml
name: Deploy to GitHub Pages

on:
  push:
    branches: [ main ]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
    - uses: actions/checkout@v4
    - uses: actions/setup-python@v5
      with:
        python-version: 3.x
    - run: pip install -r requirements.txt
    - run: mkdocs gh-deploy --force
```

## GitLab einrichten

### Konto erstellen

1. **Besuchen**: https://gitlab.com
2. **Registrieren**: E-Mail, Passwort, Benutzername
3. **E-Mail bestätigen**

### Repository erstellen

1. GitLab starten → New project
2. Projekt-Name: `mein-projekt`
3. Beschreibung: Kurze Beschreibung
4. Öffentlich/Privat wählen
5. Initialize repository with README
6. Create project

### GitLab CI/CD

**Datei**: `.gitlab-ci.yml`

```yaml
stages:
  - test
  - build
  - deploy

test:
  stage: test
  image: python:3.11
  script:
    - pip install -r requirements.txt
    - python -m pytest

build:
  stage: build
  image: node:18
  script:
    - npm install
    - npm run build
  artifacts:
    paths:
      - dist/

deploy:
  stage: deploy
  image: alpine:latest
  script:
    - apk add --no-cache rsync openssh
    - rsync -avz dist/ user@server:/path/to/deploy
  only:
    - main
```

## GitHub vs. GitLab: Vergleich

| Feature | GitHub | GitLab |
|---------|--------|--------|
| **CI/CD** | GitHub Actions | Integriert (.gitlab-ci.yml) |
| **Hosting** | Nur Cloud | Cloud + Self-Hosted |
| **Kosten** | Kostenlos (öffentlich) | Kostenlos (öffentlich) |
| **Enterprise** | GitHub Enterprise | GitLab Enterprise |
| **Dokumentation** | GitHub Pages | GitLab Pages |
| **Container** | GitHub Container Registry | GitLab Container Registry |
| **Security** | Dependabot, CodeQL | Integrierte Security Scanning |

## Best Practices

### 1. README.md

**Inhalt**:
- Projektbeschreibung
- Installation
- Verwendung
- Contributing Guidelines
- License

**Beispiel**:
```markdown
# Mein Projekt

## Beschreibung
Kurze Beschreibung des Projekts

## Installation
```bash
npm install
```

## Verwendung
```bash
npm start
```

## Contributing
1. Fork das Repository
2. Erstelle einen Feature-Branch
3. Commit Ihre Änderungen
4. Push zum Branch
5. Erstellen Sie einen Pull Request

## License
MIT
```

### 2. .gitignore

**Warum wichtig?**
- Verhindert, dass unnötige Dateien committet werden
- Schützt sensible Daten
- Reduziert Repository-Größe

**Beispiele**:
```gitignore
# Python
__pycache__/
*.py[cod]
*$py.class
.env

# Node.js
node_modules/
npm-debug.log
yarn-error.log

# IDE
.vscode/
.idea/
*.swp
*.swo

# OS
.DS_Store
Thumbs.db

# Build
dist/
build/
```

### 3. Branch Protection

**GitHub**:
1. Settings → Branches → Add rule
2. Branch name pattern: `main`
3. Require pull request reviews
4. Require status checks
5. Require conversation resolution

**GitLab**:
1. Settings → Repository → Protected branches
2. Protect `main` branch
3. Allow push only for maintainers
4. Require merge requests

## Zusammenfassung

**GitHub vs. GitLab**:
- GitHub: Community, Open Source, GitHub Actions
- GitLab: DevOps, Self-Hosted, integrierte CI/CD

**GitHub Features**:
- SSH-Schlüssel für sichere Authentifizierung
- GitHub Actions für Automatisierung
- GitHub Pages für Hosting
- Branch Protection für Sicherheit

**Best Practices**:
- Gute README.md
- Richtiges .gitignore
- Branch Protection
- CI/CD Pipelines
